# TODO: Implement Category-Only Flag For Main Pagination

## Problem Description

Posts with a `category` currently appear in both places:
- The main paginated archive generated by `src/generate.rs` and `src/generate_pagination_pages.rs`
- The category-specific archive generated by `src/generate_category_pages.rs`

There is no front matter flag to keep a post visible on its category page while excluding it from the main pagination. The existing `unlisted: true` behavior is too broad because it removes a post from both main pagination and category pagination.

## Proposed Solution

Add a new front matter flag named `category_only: true`.

Behavior:
- Posts with `category_only: true` should be excluded from the main paginated archive
- Those same posts should still be included in category pagination pages
- Existing behavior should remain unchanged for posts without the flag

This should be implemented in Rust where pagination inputs are filtered, rather than in Liquid templates, because page grouping and page counts are computed before rendering.

## Anti-Patterns (Optional)

- Do not reuse `unlisted` for this behavior; that would hide posts from category pages too
- Do not try to solve this only in pagination templates; that would produce incorrect page counts and navigation
- Do not make `category` itself imply exclusion from the main archive; that would be a breaking behavior change

## Implementation Plan

### Step 1: Extend main pagination filtering
- Update the main pagination filter in `src/generate.rs`
- Exclude posts where `category_only` is set to `true`
- Keep existing `unlisted` filtering intact

### Step 2: Preserve category pagination behavior
- Leave category page generation unchanged except for any needed tests
- Confirm that `category_only: true` posts still flow into `generate_category_pages`

### Step 3: Add regression coverage
- Add or update tests for:
- Posts with `category_only: true` being absent from main pagination
- Posts with `category_only: true` still appearing in category pagination
- Posts without the flag keeping existing behavior

## Validation Checkpoints (Optional)

- `cargo test generate -- --test-threads=1`
- `cargo test generate_pagination_pages -- --test-threads=1`
- `cargo test generate_category_pages -- --test-threads=1`

## Success Criteria

- [ ] `category_only: true` excludes posts from the main paginated archive
- [ ] `category_only: true` posts still appear on category pagination pages
- [ ] Existing sites behave the same when the flag is not used
- [ ] Tests cover the new filtering behavior

## Affected Components

- `src/generate.rs`
- `src/generate_pagination_pages.rs`
- `src/generate_category_pages.rs`
- `sites/*/posts/*.md` when the new flag is adopted

## Risks & Considerations

- **Filtering drift**: Main and category filtering rules can diverge further over time. Keep the behavior explicit in tests.
- **Backward compatibility**: The new flag should be opt-in only.
- **Naming**: `category_only` is clear, but if more archive types are added later, a broader visibility model may be preferable.
